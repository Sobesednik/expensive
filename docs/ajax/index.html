Expensive: CLI Domain Manager

\n\nExpensive is a tool that provides access to \n namecheap.com service via\nthe CLI. It can be used to quickly retrieve the list of registered domains, view more precise\ninformation about each of them, update host records by pointing the Apex domain to GitHub DNS\nservers to enable GitHub pages integration, and also register domains.\n\n

\n \n \n

\n\n

Installation & Setup

\n\nExpensive can be installed globally, and it has 0 dependencies. Use the following commands:\n\n

yarn global add expensive

\n

npm i -g expensive

\n\nYou will need the Namecheap account, so create one now (affiliate link, thank you ðŸ˜Š)!\n\n

\n \n \n

\n\nThen go to your Profile > Tools > API Access and apply for an API key.\nIf you don't receive with soon, contact support to help you get\naccess activated. Once the API key is ready, expensive can be initialised\nby running it, and answering configuration questions in interactive mode.\n\n

\n \n \n

\n\n

Usage

\n\nThese are the main commands:\n\n
    \n
  • expensive: show the list of domains on the account, and navigate pages (20 domains per page)
    • \n
  • expensive domain[.com][ domainN]: check if domains are available. If the TLD is not supplied, the domain will be checked in 10 top zones. Can list any number of domains separated by spaces.
    • \n
  • expensive domain.com -i: display information about the owned domain, including DNS records if using Namecheap DNS.
    • \n
  • expensive domain.com -r: register the domain name via CLI. If applicable, will try to look up the coupon to get discount.
    • \n
  • expensive domain.com -g: point domain's DNS host records to GitHub DNS to switch on GitHub Pages.
    • \n
\n\nFor more information about the usage of expensive, please consult the\n\n GitHub\nproject page, as well as its \n Wiki.\n\n
